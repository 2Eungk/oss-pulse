# Security Policy

Please do not open public issues for vulnerabilities.

Report vulnerabilities privately through [GitHub Private Vulnerability Reporting](https://github.com/2Eungk/oss-pulse/security/advisories/new). Do not include sensitive details in a public issue.

Include:

- affected version or commit
- reproduction steps
- expected impact
- any safe workaround

This early project does not yet have a fixed response SLA, but verified reports will be acknowledged before public disclosure.
